# 약사 시급 리포트 — 웹 버전 구현 핸드오프

이 폴더는 디자인 시안(Design Component, `.dc.html`)을 **웹 버전으로 구현**하기 위한 소스입니다.
모바일 버전은 범위에서 제외합니다.

## 구현 대상 (2개 화면)

1. **`약사 시급 리포트 (웹).dc.html`** — 메인(홈) 페이지
   - 상단 내비 + 소개문 + "분석 공고 수 3,444개" 콜아웃
   - 토글 2개(세부 분류 / 그래프) — 펼침·접힘 동작
   - 2단 레이아웃: (좌) Last Update + 컬러 시급 리스트 5종 + 지역별 평균 시급 비교(칩 선택 → 막대 하이라이트), (우) "어떤 정보가 궁금하세요?" 컬러 갤러리 카드 4종
   - **"풀타임 근무" 카드 클릭 → `풀타임 근무 상세 (웹).dc.html`로 이동**

2. **`풀타임 근무 상세 (웹).dc.html`** — 풀타임 상세 페이지
   - 소개 / 평균 시급 콜아웃 / 근무 스케줄 5종
   - 차트 6종 + 토글 안 차트 8종(전국·지역별 별표 비교 산점도, 전체 히스토그램·산점도)
   - 토글: 활용법(전국/지역별·중첩 토글), 경기 중부/외곽 구분, IQR, 참고

## 기술 메모

- 각 `.dc.html`은 **Design Component** 포맷입니다. `<x-dc>…</x-dc>` 안의 마크업이 템플릿, `<script>` 안 `class Component extends DCLogic` 가 로직이며, 런타임은 같은 폴더의 **`support.js`** 가 처리합니다. 브라우저에서 파일을 바로 열면 렌더됩니다.
- 일반 웹앱(React/Vue/순수 HTML 등)으로 다시 구현할 경우:
  - **스타일은 전부 인라인** 입니다. 템플릿 마크업 + 인라인 스타일을 그대로 옮기면 픽셀 단위로 재현됩니다.
  - **상태/인터랙션**은 로직 클래스의 `state`와 `renderVals()`에 정리돼 있습니다(토글 open 맵, 지역 선택 region, 차트 데이터). 이것을 프레임워크의 상태로 옮기세요.
  - 폰트: **Pretendard** (CDN `cdn.jsdelivr.net/gh/orioncactus/pretendard`), 브랜드 컬러: 약국 그린 `#138255` / 카테고리 파랑 `#2E6FB0`·앰버 `#C5891B`·그린 `#3C9A5C`·퍼플 `#7B5BC4`.

## ⚠️ 차트는 mock 데이터입니다 — 실데이터 교체 지점

모든 그래프는 시드 고정 난수로 생성한 **mock SVG**입니다. 차트 생성 로직은 **데이터 입력과 분리된 순수 함수**로 유지해 주세요. 실제 앱에서는 이 함수들의 입력만 실데이터로 바꾸면 됩니다.

| 함수 | 역할 | 입력(실데이터로 교체) |
|---|---|---|
| `buildHistogram(hist)` | 주당 근무시간 히스토그램 | `[[시간, 공고수], …]` |
| `buildScatter(pts, mode)` | 시급/월급/지역 산점도 (`'all'`/`'month'`/`'region'`) | `[{x:근무시간, y:시급, month, region, col}, …]` |
| `buildStrip(groups, ymax)` | 지역별 분포(IQR) 그래프 | `[{nm, mn:평균, n, vals:[{y, j}]}, …]` |
| `buildMyScatter(region, mean)` | "나의 시급 비교" 별표 산점도 | 위 `pts` 필터 + 지역 평균 |
| `buildFullHistogram(fh)` | 전체(파트+풀타임) 히스토그램 | `[시간1공고수, …]` (1~60h) |
| `buildFullScatter(pts)` | 전체 근무시간-시급 산점도 | `[{x, y}, …]` |

`getData()` 안의 하드코딩 배열(공고 수, 지역 평균, 회귀계수 등)이 실데이터로 대체될 자리입니다.
